throttle节流

函数如下，调用方式: throttle(this.search,1000,1)(),其中this.search为本地的方法

/** * @desc 函数节流 * @param func 函数 * @param wait 延迟执行毫秒数 * @param type 1 表时间戳版，2 表定时器版 */ const throttle = (func, wait ,type) => { let previous = 0; let timeout; return function() { let context = this; let args = arguments; if(type===1){ let now = Date.now(); if (now - previous > wait) { func.apply(context, args); previous = now; } }else if(type===2){ if (!timeout) { timeout = setTimeout(() => { timeout = null; func.apply(context, args) }, wait) } } } }

节流的意思，就是多少秒内，只允许执行一次。常用于监控页面滚动，鼠标不断点击事件

debounce防抖

复杂版本防抖函数,参数分别为,方法名字，延时的毫秒数,是否立即执行(true的话，就不延时了，false延时),调用方式: debounce(this.search,1000,true)(),此处this.search是本地的方法

`
    const debounce = (fn,delay,immediate) => {
        let timeout;
        return function() {
            let context = this;
            let args = arguments;
            if (timeout) clearTimeout(timeout);
            if (immediate) {
                var callNow = !timeout;
                timeout = setTimeout(() => {
                    timeout = null;
                }, delay)
                if (callNow) fn.apply(context, args)
            }
            else {
                timeout = setTimeout(()=>{
                    fn.apply(context, args)
                }, delay);
            }
        }
    }
`

简化版防抖,参数分别为，方法名字，延时毫秒数,调用方式,debounce(this.search,1000)()

`
    const debunce = (fn,delay) => {
        let timer = null
        if(timer){
            clearTimeout(timer)
        }
        timer = setTimeout(fn,delay)
    }
`

防抖的意思是，触发后，多少秒后再执行。举个例子，就像电梯，上来一个人，电梯就会在比如5S后启动，然后在这5S内，又上来一个人，那么电梯就重新计时5S后启动。比较常用与监控页面大小变化或者输入监控

computed计算属性的用法区别

如果是不传参（比如计算属性名字是getPhone），调用方式 getPhone

`
    computed:{
        getPhone(){
            return '一系列操作'
        }
    }
`

其实不传参，有个默认的参数,相当于$event，调用方式 getPhone

`
    computed:{
        getPhone(e){
            e则为 $event
            return '一系列操作'
        }
    }
`

传参的情况,比如参数是phone，就是你传的参数，调用方式 getPhone(phone)

`
    computed:{
        getPhone(){
            return (phone) =>{
                return '一系列操作'
            }
        }
    }
`
