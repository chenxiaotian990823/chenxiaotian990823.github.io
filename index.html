variable变量

全局变量var,也是顶层变量

ES5中常用，全局作用域，声明一次，在项目中任何地方都可以用，并且都可以修改，不存在暂时性死区问题
存在变量提升
`
    console.log(aaa)        //undefined
    var aaa = 1
`
为什么会打印undefined?原因如下，编译器把上述代码当成了这样执行:
`
    var aaa 
    console.log(aaa)
    aaa = 1
`
使用var，我们可以进行多次声明，后边声明的会覆盖掉之前的
`
    var b = 1
    console.log(b)          //1
    var b = 2
    console.log(b)          //2
`
顶层对象，在浏览器环境中指window，在Node中指global
`
    var a = 1
    console.log(window.a)   //1
`
如果在函数中使用使用var声明变量时候，变量是局部的
`
    var a = 20
    function change(){
        var a = 30
    }
    change()
    console.log(a) // 20
`
而如果在函数内不使用var，该变量是全局的
`
    var a = 20
    function change(){
        a = 30
    }
    change()
    console.log(a) // 30
`

let，let是局部变量，用法类似于var，但只作用于自己的作用域

如果在作用域外使用，则会报错
`
    {
        let b = 1
    }
    console.log(b)   // ReferenceError: b is not defined.
`
不存在变量提升
`
    console.log(a)  //ReferenceError
    let a = 2
`
使用let声明变量前，该变量都不可用，也就是大家常说的“暂时性死区”
let不允许在相同作用域中重复声明
`
    let a = 20
    let a = 30
    // Uncaught SyntaxError: Identifier 'a' has already been declared
`
我们不能在函数内部重新声明参数
`
    function func(arg) {
        let arg;
    }
    func()
    // Uncaught SyntaxError: Identifier 'arg' has already been declared
`

const

const声明一个只读的常量，一旦声明，常量的值就不能改变
`
    const a = 1
    a = 3
    // TypeError: Assignment to constant variable.
`
这意味着，const一旦声明变量，就必须立即初始化，不能留到以后赋值
`
    const a;
    // SyntaxError: Missing initializer in const declaration
`
如果之前用var或let声明过变量，再用const声明同样会报错
`
    var a = 20
    let b = 20
    const a = 30
    const b = 30
    // 都会报错
`
const实际上保证的并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动
对于简单类型的数据，值就保存在变量指向的那个内存地址，因此等同于常量
对于复杂类型的数据，变量指向的内存地址，保存的只是一个指向实际数据的指针，const只能保证这个指针是固定的，并不能确保改变量的结构不变
`
    const foo = {};

    // 为 foo 添加一个属性，可以成功
    foo.prop = 123;
    foo.prop // 123

    // 将 foo 指向另一个对象，就会报错
    foo = {}; // TypeError: "foo" is read-only
`
其它情况，const与let一致

区别

变量提升

var声明的变量存在变量提升，即变量可以在声明之前调用，值为undefined
let和const不存在变量提升，即它们所声明的变量一定要在声明后使用，否则报错
`
    // var
    console.log(a)  // undefined
    var a = 10

    // let 
    console.log(b)  // Cannot access 'b' before initialization
    let b = 10

    // const
    console.log(c)  // Cannot access 'c' before initialization
    const c = 10
`

暂时性死区

var不存在暂时性死区
let和const存在暂时性死区，只有等到声明变量的那一行代码出现，才可以获取和使用该变量
`
    // var
    console.log(a)  // undefined
    var a = 10

    // let
    console.log(b)  // Cannot access 'b' before initialization
    let b = 10

    // const
    console.log(c)  // Cannot access 'c' before initialization
    const c = 10
`

块级作用域

var不存在块级作用域
let和const存在块级作用域
`
    // var
    {
        var a = 20
    }
    console.log(a)  // 20

    // let
    {
        let b = 20
    }
    console.log(b)  // Uncaught ReferenceError: b is not defined

    // const
    {
        const c = 20
    }
    console.log(c)  // Uncaught ReferenceError: c is not defined
`

重复声明

var允许重复声明变量
let和const在同一作用域不允许重复声明变量
`
    // var
    var a = 10
    var a = 20 // 20

    // let
    let b = 10
    let b = 20 // Identifier 'b' has already been declared

    // const
    const c = 10
    const c = 20 // Identifier 'c' has already been declared
`

修改声明的变量

var和let可以
const声明一个只读的常量。一旦声明，常量的值就不能改变
`
    // var
    var a = 10
    a = 20
    console.log(a)  // 20

    //let
    let b = 10
    b = 20
    console.log(b)  // 20

    // const
    const c = 10
    c = 20
    console.log(c) // Uncaught TypeError: Assignment to constant variable
`

使用

能用const的情况尽量使用const，其他情况下大多数使用let，避免使用var

currying柯里化函数

我们平常写方法，比如写个加法，调用方式，this.add(4,5)

`
    add(a,b){
        return a + b
    }
`

使用currying后呢，我们可以写成,把原来的同时传两个参数，分开了，调用方式:

`
    add(a){
        return (b)=>{
            return a + b
        }
    }
`

这么看上去好像没有必要，但是有时候我们比如封装了一个方法,但是我们有其中一个参数是固定的，比如我们总要算5+任意数

`
    add(a){
        return (b) =>{
            return a + b
        }
    }

    let result = add(5)
    console.log(result(6))      //11
`

注:这样就可以封装一些常用的方法，比如正则表达式这种，总要验证手机号啥的，就可以类似封装

这只是简单用法，详细用法可以参考一下 https://www.jianshu.com/p/2975c25e4d71

promise基本用法

promise 封装在方法内,比如方法为getPhone，可以进行传参，然后进行一些判断

`
    getPhone(e){
        let p = new Promise(() =>{
            if(e == 0){
                resolve('成功')
            }else{
                reject('失败')
            }
        })
        return p
    },
`

以上的调用方式:,其中resolve回来的参数，在res中，reject中的参数，在err中

`
    this.getPhone(0)
    .then(res =>{
        console.log(res)
    })
    .catch(err =>{
        console.log(err)
    })
`

promise.all，当包含的所有promise中执行完，再进行操作,注:有一个失败就算失败，两个都成功才算成功

`
    let result = Promise.all([this.getPhone(0),this.getPhone(1)])
    result.then(res =>{
        console.log(res)
    })
    .catch(err =>{
        console.log(err)
    })
`

throttle节流

函数如下，调用方式: throttle(this.search,1000,1)(),其中this.search为本地的方法

`
    const throttle = (func, wait ,type) => {
        let previous = 0;
        let timeout;
        return function() {
            let context = this;
            let args = arguments;
            if(type===1){
                let now = Date.now();
                if (now - previous > wait) {
                    func.apply(context, args);
                    previous = now;
                }
            }else if(type===2){
                if (!timeout) {
                    timeout = setTimeout(() => {
                        timeout = null;
                        func.apply(context, args)
                    }, wait)
                }
            }
        }
    }
`

节流的意思，就是多少秒内，只允许执行一次。常用于监控页面滚动，鼠标不断点击事件

debounce防抖

复杂版本防抖函数,参数分别为,方法名字，延时的毫秒数,是否立即执行(true的话，就不延时了，false延时),调用方式: debounce(this.search,1000,true)(),此处this.search是本地的方法

`
    const debounce = (fn,delay,immediate) => {
        let timeout;
        return function() {
            let context = this;
            let args = arguments;
            if (timeout) clearTimeout(timeout);
            if (immediate) {
                var callNow = !timeout;
                timeout = setTimeout(() => {
                    timeout = null;
                }, delay)
                if (callNow) fn.apply(context, args)
            }
            else {
                timeout = setTimeout(()=>{
                    fn.apply(context, args)
                }, delay);
            }
        }
    }
`

简化版防抖,参数分别为，方法名字，延时毫秒数,调用方式,debounce(this.search,1000)()

`
    const debunce = (fn,delay) => {
        let timer = null
        if(timer){
            clearTimeout(timer)
        }
        timer = setTimeout(fn,delay)
    }
`

防抖的意思是，触发后，多少秒后再执行。举个例子，就像电梯，上来一个人，电梯就会在比如5S后启动，然后在这5S内，又上来一个人，那么电梯就重新计时5S后启动。比较常用与监控页面大小变化或者输入监控

computed计算属性的用法区别

如果是不传参（比如计算属性名字是getPhone），调用方式 getPhone

`
    computed:{
        getPhone(){
            return '一系列操作'
        }
    }
`

其实不传参，有个默认的参数,相当于$event，调用方式 getPhone

`
    computed:{
        getPhone(e){
            e则为 $event
            return '一系列操作'
        }
    }
`

传参的情况,比如参数是phone，就是你传的参数，调用方式 getPhone(phone)

`
    computed:{
        getPhone(){
            return (phone) =>{
                return '一系列操作'
            }
        }
    }
`

计算属性与methods方法的区别，计算属性有自己的缓存，相同参数只会调用一次，而methods不会，每次都要调用。所以计算属性性能更好
